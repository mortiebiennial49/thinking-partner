## 🚀 What this is

thinking-partner is a skill pack for AI agents. It helps agents think in a more useful way by using mental models, orientation checks, and cognitive operations.

It works with tools like:

- Claude Code
- Cursor
- Windsurf
- Cline
- GitHub Copilot

Use it when you want an agent to:

- compare options
- spot weak assumptions
- stay on task
- break down hard problems
- think in a more structured way

## 🪄 Why use it

AI agents can move fast, but speed can lead to shallow answers. thinking-partner gives the agent a set of mental tools it can use during planning, review, and decision-making.

It helps with:

- clearer reasoning
- better tradeoffs
- stronger problem framing
- fewer missed steps
- more careful decision paths

The goal is not to make the agent sound smart. The goal is to help it think in a clean, useful way.

## 🧭 What is inside

thinking-partner is built around three main parts:

### 1. Mental models

These are simple ways to look at a problem. They help the agent choose a useful frame before giving an answer.

Examples:

- first principles
- cost and benefit
- tradeoff checks
- cause and effect
- edge case review
- second-order effects

### 2. Orientation detection

This helps the agent understand what kind of task it is facing.

It can tell the difference between things like:

- a decision
- a plan
- a review
- a diagnosis
- a comparison
- a cleanup task

That matters because each task needs a different kind of thinking.

### 3. Cognitive operations

These are the thinking steps the agent can use while working.

Examples:

- break down a goal
- test an assumption
- compare options
- rank risks
- check for missing facts
- narrow the scope
- build a step-by-step path

## 🛠️ How to use it

After you install it, use it inside your agent tool the same way you would use other skills or helper packs.

Try prompts like:

- Review this answer for missing steps

The skill works best when you give the agent a clear goal and a real task. It can then apply the right mental model instead of guessing.

## 🔍 When it helps most

Use thinking-partner when the task has more than one possible answer.

Good cases include:

- picking between tools or plans
- reviewing a draft
- finding weak points in a proposal
- deciding what to do first

It is less useful for very small tasks that need one short answer.
